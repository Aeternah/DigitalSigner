# DigitalSigner

# Учебно-исследовательская работа

## Разработка настольного приложения для создания и проверки электронной цифровой подписи

---

## Описание проекта

**DigitalSigner** — настольное приложение с графическим интерфейсом, предназначенное для генерации RSA-ключей, создания и проверки электронной цифровой подписи файлов, а также вычисления SHA-256 хеша.

Приложение разработано на языке **C++** с использованием фреймворка **Qt5** и криптографической библиотеки **OpenSSL**.

Цель проекта — изучение принципов работы современных криптографических алгоритмов и разработка программного средства для обеспечения целостности и подлинности данных.

---

## Основные возможности

* Генерация RSA-ключей:

  * RSA-2048;
  * RSA-3072;
  * RSA-4096.
* Защита приватного ключа паролем.
* Создание электронной цифровой подписи файлов.
* Проверка корректности электронной подписи.
* Вычисление SHA-256 хеша файлов.
* Просмотр содержимого файлов подписи.
* Ведение журнала выполнения операций.
* Безопасная очистка конфиденциальных данных из оперативной памяти.

## Принцип работы приложения

### Генерация ключей

Пользователь выбирает размер ключа и место сохранения файлов.

После генерации создаются:

* приватный ключ (`private.pem`);
* публичный ключ (`public.pem`).

---

### Подписание файла

1. Пользователь выбирает файл.
2. Выбирает приватный ключ.
3. При необходимости вводит пароль.
4. Приложение вычисляет SHA-256 хеш.
5. Полученный хеш подписывается алгоритмом RSA.
6. Подпись сохраняется в файл `.sig`.

---

### Проверка подписи

Для проверки необходимо указать:

* исходный файл;
* файл подписи;
* публичный ключ.

Программа вычисляет SHA-256 хеш исходного файла и сравнивает его с расшифрованной подписью.

Если значения совпадают, подпись признаётся действительной.

# Сборка проекта

## Требования

* Компилятор с поддержкой C++17;
* CMake 3.16 и выше;
* Qt5;
* OpenSSL.

---

## Сборка в Ubuntu

### Установка зависимостей

```bash
sudo apt update

sudo apt install build-essential \
                 cmake \
                 qtbase5-dev \
                 libssl-dev
```

### Клонирование проекта

```bash
git clone <repository_url>
cd DigitalSigner
```

### Сборка

```bash
mkdir build
cd build

cmake ..
make -j$(nproc)
```

### Запуск

```bash
./DigitalSigner
```

---

## Сборка в Windows

### Необходимое программное обеспечение

Установите:

* Visual Studio 2022;
* компонент **Desktop development with C++**;
* Qt5;
* OpenSSL;
* CMake.

---

### Сборка через Visual Studio

Откройте терминал:

```text
x64 Native Tools Command Prompt for VS 2022
```

Выполните:

```bash
git clone <repository_url>
cd DigitalSigner

mkdir build
cd build

cmake .. -G "Visual Studio 17 2022"
cmake --build . --config Release
```

Исполняемый файл будет расположен по пути:

```text
build/Release/DigitalSigner.exe
```

Запуск:

```bash
.\Release\DigitalSigner.exe
```

---

### Сборка через Qt Creator

1. Откройте Qt Creator.
2. Выберите:

```text
File → Open File or Project
```

3. Укажите файл:

```text
CMakeLists.txt
```

4. Выберите доступный Kit (MSVC или MinGW).
5. Нажмите **Configure Project**.
6. Выполните сборку (`Ctrl + B`).
7. Запустите проект (`Ctrl + R`).
